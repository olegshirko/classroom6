## Часть 1: Рефакторинг типов данных

### Зачем

В предыдущей работе ты использовал `dict` для хранения всех данных. Это работает, но не всегда эффективно. В этой части ты найдёшь узкие места и заменишь типы на более подходящие.

### Проблемы в текущем коде

**Проблема 1: Медленный поиск по email**

Сейчас `users_db` — словарь с ключом `id`, но поиск пользователя идёт по email через перебор всех значений:

```python
# Сейчас — O(n), перебираем всех пользователей
for user in users_db.values():
    if user.email == email:
        return user
```

**Задача:** Добавить второй словарь с email как ключом для поиска за O(1):

```python
# Станет — O(1), сразу находим по ключу
users_by_email: dict[str, UserInDB] = {
    "admin@shop.com": UserInDB(...),
    "customer@shop.com": UserInDB(...),
}
```

**Проблема 2: Только одна роль (admin/customer)**

Сейчас `is_admin: bool` — можно быть только админом или клиентом. А если нужен модератор, продавец, поддержка?

**Задача:** Заменить `is_admin: bool` на `roles: set[str]`:

```python
# Было
is_admin: bool = True

# Станет
roles: set[str] = {"admin", "moderator"}
```

`set` позволяет хранить несколько ролей у одного пользователя и быстро проверять наличие:

```python
if "admin" in user.roles:  # O(1) проверка
    ...
```

**Проблема 3: Нет категорий товаров**

Все товары в куче. Хочется фильтровать по категориям.

**Задача:** Добавить фиксированный набор категорий через `tuple`:

```python
CATEGORIES: tuple[str, ...] = ("Ноутбуки", "Телефоны", "Планшеты", "Аксессуары")
```

`tuple` нельзя случайно изменить — это защита от ошибок.

**Проблема 4: Одна корзина на всех**

Сейчас `cart_db` общая для всех авторизованных пользователей. У каждого должна быть своя корзина.

**Задача:** Привязать корзины к email пользователя:

```python
# Было: одна общая корзина
cart_db: dict[int, CartItemInDB] = {}

# Станет: отдельная корзина для каждого пользователя
carts: dict[str, dict[int, int]] = {
    "customer@shop.com": {1: 2, 3: 1},  # email → {item_id: quantity}
}
```

### Что нужно сделать

1. Добавить `users_by_email: dict[str, UserInDB]` и использовать его вместо перебора
2. Заменить `is_admin: bool` на `roles: set[str]` в моделях и данных
3. Обновить функцию `require_admin` на проверку `"admin" in user.roles`
4. Добавить `CATEGORIES: tuple[str, ...]` и поле `category` в `Item`/`ItemInDB`
5. Привязать корзины к email через `carts: dict[str, dict[int, int]]`
6. Обновить все существующие ручки под новые структуры
7. Все старые тесты должны проходить!

## Часть 2: Новые ручки

Выбери **3-5 ручек** из списка ниже и реализуй их. Каждая ручка требует свой тип данных — подумай, какой выбрать.

### Доступные ручки

| Ручка | Что делает | Подходящий тип |
|-------|------------|----------------|
| `GET /categories` | Список категорий | `tuple[str, ...]` |
| `GET /items?category=...` | Фильтр по категории | `dict[int, str]` — id → category |
| `POST /items/{id}/rate` | Оценить товар (1-5) | `dict[int, list[int]]` — id → оценки |
| `GET /items/{id}/rating` | Средний рейтинг товара | `dict[int, float]` — id → средний балл |
| `GET /items/popular` | Топ-5 по просмотрам | `list[int]` — ID в порядке просмотров |
| `POST /wishlist/add/{id}` | Добавить в избранное | `dict[str, set[int]]` — email → {item_ids} |
| `GET /wishlist` | Моё избранное | `dict[str, set[int]]` |
| `DELETE /wishlist/remove/{id}` | Убрать из избранного | `set[int]` |
| `GET /items/search/history` | История моих поисков | `dict[str, list[str]]` — email → запросы |

### Требования к новым ручкам

- Каждая ручка должна работать с правильным типом данных
- Добавить модели Pydantic для запросов/ответов
- Написать тесты на новые ручки (минимум 2 теста на ручку)
- Авторизация где нужна (wishlist, история поиска)

## Запуск

```bash
# Создать виртуальное окружение
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt

# Запустить
uvicorn main:app -reload
```

После запуска открой в браузере:
- `http://localhost:8000/items` - список товаров
- `http://localhost:8000/categories` - категории (после реализации)
