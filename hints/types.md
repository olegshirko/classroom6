# Шпаргалка: Типы данных в Python

Когда использовать `dict`, `set`, `tuple`, `list` — и почему это важно.

---

## dict — быстрый поиск по ключу

**Когда:** нужен быстрый доступ к данным по идентификатору (ключу).

**Сложность:** O(1) — поиск, вставка, удаление.

```python
# Поиск пользователя по email — O(1)
users_by_email: dict[str, UserInDB] = {
    "admin@shop.com": UserInDB(id=1, name="Админ", ...),
    "customer@shop.com": UserInDB(id=2, name="Клиент", ...),
}

# Вместо перебора всех пользователей:
# for user in users_db.values():  # O(n) — медленно при большом количестве
#     if user.email == email:
#         return user

# Сразу находим:
user = users_by_email.get("admin@shop.com")  # O(1) — быстро!
```

**Ещё примеры:**
- `item_by_id: dict[int, ItemInDB]` — товары по ID
- `carts: dict[str, dict[int, int]]` — корзины пользователей (email → {item_id: quantity})
- `ratings: dict[int, list[int]]` — оценки товаров (item_id → список оценок)

---

## set — уникальность и быстрая проверка

**Когда:** нужно хранить набор уникальных значений или быстро проверять наличие.

**Сложность:** O(1) — проверка наличия, добавление, удаление.

```python
# Роли пользователя — можно иметь несколько
roles: set[str] = {"admin", "moderator"}

# Проверка роли — O(1)
if "admin" in roles:
    print("Доступ разрешён")

# Вместо is_admin: bool — только одно значение
# is_admin = True  # а если нужен ещё модератор?
```

**Ещё примеры:**
- `wishlist: set[int]` — ID товаров в избранном (без дублей)
- `banned_emails: set[str]` — забаненные пользователи
- `search_results: set[int]` — уникальные ID найденных товаров

**Операции с множествами:**
```python
a = {1, 2, 3}
b = {3, 4, 5}

a | b  # объединение: {1, 2, 3, 4, 5}
a & b  # пересечение: {3}
a - b  # разница: {1, 2}
```

---

## tuple — неизменяемость

**Когда:** данные не должны меняться после создания. Защита от случайного изменения.

**Сложность:** O(1) — доступ по индексу. Нельзя добавить/удалить элемент.

```python
# Категории товаров — фиксированный список
CATEGORIES: tuple[str, ...] = ("Ноутбуки", "Телефоны", "Планшеты", "Аксессуары")

# Нельзя случайно изменить:
# CATEGORIES.append("Новое")  # AttributeError!

# Но можно перебрать:
for category in CATEGORIES:
    print(category)
```

**Ещё примеры:**
- `PRICE_TIERS: tuple[tuple[float, float], ...]` — ценовые диапазоны
- `BANNER_SIZE: tuple[int, int] = (1200, 400)` — размеры изображения
- `ADMIN_EMAILS: tuple[str, ...]` — список админов (не меняется)

---

## list — порядок важен

**Когда:** нужен порядок элементов, возможны дубликаты, элементы добавляются/удаляются.

**Сложность:** O(1) — доступ по индексу, O(n) — поиск, вставка в начало/середину.

```python
# История поиска — важен порядок (последние запросы)
search_history: list[str] = ["айфон", "макбук", "айпад"]

# Последний запрос:
last_query = search_history[-1]  # "айпад"

# Добавить в конец:
search_history.append("эйрподс")
```

**Ещё примеры:**
- `operation_log: list[dict]` — лог действий (хронологический порядок)
- `popular_items: list[int]` — ID товаров по популярности
- `ratings: list[int]` — оценки товара `[5, 4, 5, 3, 5]`

---

## Сравнение типов

| Тип | Изменяемый | Уникальность | Порядок | Быстрый поиск |
|-----|------------|--------------|---------|---------------|
| `list` | ✅ | ❌ | ✅ | O(n) |
| `tuple` | ❌ | ❌ | ✅ | O(1) по индексу |
| `set` | ✅ | ✅ | ❌ | O(1) |
| `dict` | ✅ | ключи ✅ | ✅ (с Python 3.7) | O(1) по ключу |

---

## Как выбрать тип

Задай себе вопросы:

1. **Нужен ли быстрый поиск по ключу?** → `dict`
2. **Нужна ли уникальность элементов?** → `set`
3. **Данные не должны меняться?** → `tuple`
4. **Важен ли порядок элементов?** → `list` или `tuple`
5. **Нужно ли часто добавлять/удалять?** → `list` или `set`

## Комбинации типов

Часто нужно вкладывать одни типы в другие:

```python
# Корзины пользователей: email → {item_id: quantity}
carts: dict[str, dict[int, int]] = {
    "customer@shop.com": {1: 2, 3: 1},
    "admin@shop.com": {2: 1},
}

# Оценки товаров: item_id → список оценок
ratings: dict[int, list[int]] = {
    1: [5, 4, 5, 3],
    2: [5, 5, 4],
}

# Избранное пользователей: email → set[item_id]
wishlist: dict[str, set[int]] = {
    "customer@shop.com": {1, 3, 4},
}

# Средний рейтинг: item_id → средний балл
avg_ratings: dict[int, float] = {
    1: 4.25,  # (5+4+5+3)/4
    2: 4.67,  # (5+5+4)/3
}
```

## Полезные функции

```python
# dict
users_by_email.get("admin@shop.com")  # None если не найден
users_by_email.keys()                 # все ключи
users_by_email.values()               # все значения

# set
roles.add("moderator")                # добавить элемент
roles.remove("admin")                 # удалить (ошибка если нет)
roles.discard("admin")                # удалить (без ошибки)
"admin" in roles                      # проверка наличия

# tuple
CATEGORIES[0]                         # доступ по индексу
len(CATEGORIES)                       # количество элементов

# list
search_history.append("запрос")       # добавить в конец
search_history.pop()                  # удалить последний
search_history[-3:]                   # последние 3 элемента
len(search_history)                   # количество элементов

# Среднее значение списка
ratings = [5, 4, 5, 3]
avg = sum(ratings) / len(ratings)     # 4.25
```
